关于我（1）-为什么要写博客？

发表于 2021-11-08 更新于 2021-12-29 分类于 3 Self ， Individual 阅读次数： Disqus：
本文字数： 2.8k 阅读时长 ≈ 5 分钟

这是我的第一篇博客，我决定从“我”开始讲起。为什么选择以这种方式开启我的博客之路呢？这就与我对写博客这件事儿的认识以及我在其上寄予的愿景息息相关了。接下来，听我慢慢讲。

在我读中学的时候，我就发现自己有一种不知道是好是坏的“癖好”——我很喜欢进行系统地整理。因此我会做出工工整整的各科笔记，以几乎相同的格式把每个章节的知识完整地记录下来，并配上目录，甚至会为此选用制式相同的本子和笔。这个“癖好”还表现在很多方面，比如当我读到一些我以为的好书，或者看到一些好的电影时，我会把它们都集中记录下来，最开始记录在纸上，后来记录在豆瓣，再后来是在网盘和硬盘里…

为什么说不知道是好是坏呢？一方面，在某些方面，这个“癖好”给我的帮助确实不小。比如和中、高考打交道的那几年，这个“癖好”让我有了相对更为扎实的基础；对于一本值得反复阅读的好书，或者一段值得反复观看的影片，当我再次回忆起它们时，不至于遍寻脑海而毫无头绪。但另一方面，这个“癖好”带来的问题也是显而易见的——它太浪费时间了。做到整齐划一大概是这个世界上最难的事儿之一了，更何况，在规整地记录之后，更多地想法会随之涌现出来，比如分类，这又是一大难题。在这个过程中，为了做到强迫症般的整齐，自然少不了各种各样的形式主义，时间也就如流水般逝去了。

即便如此，这个“癖好”我还是一直保留到了现在，因为我实在从中获益良多。当然，现在我会有意识地在践行的过程中减少形式主义的东西，让它回归初心时的样子。为什么要专门提到这样一个“癖好”呢？因为我搭建博客最初的动力，就与此有关。

我学的是计算机，进大学之前，是典型的“小镇做题家”，对这个专业只知道目光所及的前景和自认为的兴趣。进大学第一年，学校开的都是些公共课，数学、英语、体育等等，临到这年结束，才开了一门儿专业课，叫“C++高级语言程序设计”。五十多的副院长，操着一口听不懂哪儿的方言，拿着自己编的课本儿，用着将近二十年前的PPT，教我们写C风格的旧版本C++。我大概听了两节课，心里就一片灰暗，那时候还没有考虑什么大厂、什么保研，只是单纯觉得，要是这样学四年下来，我什么都学不会，而且时间全部会浪费掉。于是从那儿起，我开始摸索我的计算机自学之路。

而在摸索过程中，就不可避免的会接触到各种各样的博客。从最初通过百度接触到的博客平台CSDN、博客园，到一些名声在外的独立博客网站“CoolShell”、“阮一峰的网络日志”，再到通过Google接触到的各种各样的独立博客xxx.github.io等等。这还仅仅是中文博客，英文世界里还有更多大神的博客等待着被探索。各式各样的人在博客上记录自己的技术之路，这些内容给到摸索中的我很大的帮助，所以，我的“癖好”又被勾起来了。

没多久之后，我脑子里就一直在想，我是不是也可以自己写一个博客，把自己自学计算机技术的整个过程全部记录下来分享出去。基于这个念头，我又去了解了很多和博客相关的事儿，知道了在应聘时，一个持续更新的博客形成的优势不亚于项目和实习；知道了在未来，一个运营良好的优质博客甚至可以成为程序员的另一条出路。于是在这种种想法的推动下，我终于开始着手搭建自己的博客。

要搭建怎样的博客呢？博客平台似乎还不错，CSDN虽然广告泛滥，但博客园还算说得过去，起初我这样想。但后来，随着了解得更深入，我发现从应聘的角度讲，不同类型的博客之间也存在高下之分。就像在简历上写Gmail邮箱就是比QQ邮箱给人印象更好一样，独立博客相对于博客平台也有这样的优势。另外，GFW也是我不得不考虑的因素。我不是只知道唱红歌的乖孩子，相较于粉饰出来的表象，我对真实的现实更感兴趣，所以也希望以后能在自己的博客上有自由表达的权利，想说的时候能说两句实话。综合考虑这些因素，最终我决定搭建独立博客，并在一番权衡后，选择先通过Hexo + GitHub Pages这样便宜简洁的方式来搭建，最终的成果就是你现在看到的这个博客。

讲了半天，总算是把“为什么要搭建博客”这个事儿说清楚了，接下来就可以讲“为什么要写博客”了。

有人可能要问了，你刚刚不是说过搭建博客就是为了记录、分享自学计算机的全过程吗？既然主题都已确定，搭建完博客直接写不就好了嘛，何必单独拿出来讲呢？这话倒是没错，可问题恰恰就出在这儿。

对于那些影视爱好者来讲，他们收藏自己喜欢的影片的方法有很多：可以只是列一个清单，可以是专门买一个硬盘，甚至可以制作专业的碟片，不管选择哪种方式，最终目的都是把那些自己心爱的影片收藏起来。至于你心爱的影片是什么？和你如何收藏它无关。因为心爱，所以收藏。

搭建博客和写博客之间的关系也是这样。我把博客搭建好了以后，只是为我可能写出的内容选择了博客这样的一种呈现方式。那么，我为什么要写这些内容呢？真的只是为了记录、分享自己自学计算机的过程吗？坦白讲，在我搭建好博客后，一时的热情褪去，真的要开始下笔写博客的时候，我重新问了自己这个问题，得到的答案是否定的。如果仅仅以此为动力，我想它不足以支持我把这个博客一直写下去。

问题出在哪儿？我认真思考着这件事儿。

以“记录、分享自己自学计算机的过程”为动力的话，看在这个博客对以后求职利好的份儿上，我想我能够坚持去更新它，但既然是坚持，那么过程必定痛苦，很难长久。相较于坚持去做一件事儿，我更喜欢的还是以兴趣为引子，自发地去做一件事儿，自发慢慢形成习惯，就变成了自律，这个过程与坚持相仿，但并不会痛苦，反而滋味丰富，这样才能够提供持久的动力。那么我对计算机没兴趣吗？当然有兴趣…这，似乎陷入了一个死循环。

就这么纠结了好一阵子之后，有一天我突然反应过来，问题的症结根本不在这儿——我搭建博客这个事儿最早其实就是源于那么一个小“癖好”，可以说是临时起意，要搭建一个计算机主题的博客。但当我真的靠着三分钟热度把这个博客搭建了起来以后，要开始写博客了，我需要为自己写博客这件事儿找到长期持久的动力，仅靠这么个小“癖好”无法支撑了，那么我的动力要从哪儿来呢？

这个时候，我没有选择去叩问自己的内心，而是选择了完全听从“别人”的意见。“学计算机的就该维护一个自己的博客…”、“一个维护的好的博客在求职的时候是绝对的加分项…”、“…”等等，这些全都是“别人”说的。我从来不曾问过自己，“你为什么要写一个计算机主题的博客？”要知道，对计算机有兴趣并不意味着就能够维护一个计算机主题的博客，这完全是两码事儿。更进一步讲，我要写的难道真的是一个计算机主题的博客吗？也许，我真正想记录的其实是自己的生活，而计算机只是其中的一部分？如果是这样的话，那它是怎样的一部分呢？又或者，其实我根本就不喜欢记录…

总而言之，写不写？写什么？为什么要写？这三件事儿才是症结所在，把这三个事儿搞明白了，问题自然迎刃而解。而要想搞明白它们，得去叩问自己的内心。至于”别人“的话，最多只能作为指引方向的参照物，毕竟，最终想要到达的目的地，始终藏在自己心里。
